# petclinic-demo ‚ÄĒ TIA applied to a real black-box suite

A worked, **end-to-end** example of Test-Impact-Analysis run against the out-of-process
REST Assured black-box suite of [`baekchangjoon/spring-petclinic`][petclinic] (24 test cases,
5 `‚Ä¶BlackBoxIT` classes). Everything here is real measurement ‚ÄĒ no mock data.

## What it shows

| View | Artifact |
|---|---|
| Interactive report (5 tabs) | [`report.html`](report.html) ‚ÄĒ open in any browser |
| Executive summary + caveats | [`00-SUMMARY.md`](00-SUMMARY.md) |
| Per-test impact range, reverse index, hotspots, blind spots | [`01-coverage-report.md`](01-coverage-report.md) |
| `tia impact` on 7 real diffs | [`02-impact-scenarios.txt`](02-impact-scenarios.txt) / `scenarios.json` |

Headline results: a one-line REST-controller change selects **1‚Äď3 / 24** tests (87‚Äď96 %
reduction); the suite is flaky-free across 5 runs; and only **15 / 40** production files are
touched by the black-box suite (the MVC/Thymeleaf layer and security infra are blind spots).

## Pipeline

```
petclinic bootJar (JDK21) ‚ĒÄ‚Ėļ running app ‚óĄ‚ĒÄ -javaagent: jacocoagent-parallel.jar (port 6310)
        ‚Ė≤                                       ‚Ēā 1 vanilla-JaCoCo .exec per test
   ./gradlew blackboxTest (REST Assured + baggage: test.id) ‚Ēė
        ‚Ēā   /tmp/petclinic-coverage/<Class#method>.exec √ó24
        ‚Ėľ   exec_to_testwise.py ‚ĒÄ‚Ėļ testwise.json
   tia index ‚Üí tia.db   |   tia impact ‚Üí selection   |   tia flaky ‚Üí ratio   |   make_html.py ‚Üí report.html
```

## Reproduce

```bash
bash petclinic-demo/run-petclinic-tia.sh   # build ‚Üí run ‚Üí collect ‚Üí index ‚Üí impact ‚Üí flaky ‚Üí HTML
open  petclinic-demo/report.html
```

Prereqs: `spring-petclinic` and [`parallel-per-test-coverage`][agent] cloned locally, JDK 21 + 17,
and `org.jacoco.cli:*:nodeps` in `~/.m2`. Paths are overridable via `PETC` / `AGENT_REPO` env vars.
The script perturbs the petclinic tree only transiently (perturb-and-revert); nothing survives.

## Files

- `run-petclinic-tia.sh` ‚ÄĒ one-command orchestrator
- `exec_to_testwise.py` ‚ÄĒ bridge: per-test `.exec` ‚Üí TIA testwise JSON (via `jacococli`)
- `run_scenarios.py` ‚ÄĒ `tia impact` scenario runner ‚Üí `scenarios.json`
- `report.py` ‚ÄĒ per-test markdown report; `make_html.py` ‚ÄĒ interactive HTML
- `testwise.json` ¬∑ `scenarios.json` ¬∑ `flaky.json` ¬∑ `flaky-runs/` ¬∑ `prod-files.txt` ‚ÄĒ sample data
- `tia.db` (gitignored) ‚ÄĒ regenerated by the script
