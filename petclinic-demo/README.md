# petclinic-demo ‚Äî TIA applied to a real black-box suite

A worked, **end-to-end** example of Test-Impact-Analysis run against the out-of-process
REST Assured black-box suite of [`baekchangjoon/spring-petclinic`][petclinic] (24 test cases,
5 `‚Ä¶BlackBoxIT` classes). Everything here is real measurement ‚Äî no mock data.

## What it shows

| View | Artifact |
|---|---|
| Interactive report (5 tabs) | [`report.html`](report.html) ‚Äî open in any browser. Every column sorts asc/desc; per-test files deep-link into the JaCoCo HTML line, reverse-index tests open the local test source, the `tia impact` legend explains each term, and flaky ratios show percent + fraction. |
| Executive summary + caveats | [`00-SUMMARY.md`](00-SUMMARY.md) |
| Per-test impact range, reverse index, hotspots, blind spots | [`01-coverage-report.md`](01-coverage-report.md) |
| `tia impact` on 7 real diffs | [`02-impact-scenarios.txt`](02-impact-scenarios.txt) / `scenarios.json` |

Headline results: a one-line REST-controller change selects **1‚Äì3 / 24** tests (87‚Äì96 %
reduction); the suite is flaky-free across 5 runs; and only **15 / 40** production files are
touched by the black-box suite (the MVC/Thymeleaf layer and security infra are blind spots).

## Pipeline

```
petclinic bootJar (JDK21) ‚îÄ‚ñ∫ running app ‚óÑ‚îÄ -javaagent: jacocoagent-parallel.jar (port 6310)
        ‚ñ≤                                       ‚îÇ 1 vanilla-JaCoCo .exec per test
   ./gradlew blackboxTest (REST Assured + baggage: test.id) ‚îò
        ‚îÇ   /tmp/petclinic-coverage/<Class#method>.exec √ó24
        ‚ñº   exec_to_testwise.py ‚îÄ‚ñ∫ testwise.json
   tia index ‚Üí tia.db   |   tia impact ‚Üí selection   |   tia flaky ‚Üí ratio
        ‚îÇ   jacococli merge+report ‚Üí jacoco/ (HTML, per-test-file deep-link target)
        ‚ñº   make_html.py ‚Üí report.html
```

## Reproduce

```bash
bash petclinic-demo/run-petclinic-tia.sh   # build ‚Üí run ‚Üí collect ‚Üí index ‚Üí impact ‚Üí flaky ‚Üí HTML
open  petclinic-demo/report.html
```

Prereqs: `spring-petclinic` and [`parallel-per-test-coverage`][agent] cloned locally, JDK 21 + 17,
and `org.jacoco.cli:*:nodeps` in `~/.m2`. Paths are overridable via `PETC` / `AGENT_REPO` env vars.
The script perturbs the petclinic tree only transiently (perturb-and-revert); nothing survives.

## Files

- `run-petclinic-tia.sh` ‚Äî one-command orchestrator
- `exec_to_testwise.py` ‚Äî bridge: per-test `.exec` ‚Üí TIA testwise JSON (via `jacococli`)
- `run_scenarios.py` ‚Äî `tia impact` scenario runner ‚Üí `scenarios.json`
- `report.py` ‚Äî per-test markdown report; `make_html.py` ‚Äî interactive HTML
  (args: `‚Ä¶ <commit> <out.html> [sut_name] [jacoco_dir] [test_src_root]`)
- `jacoco/` ‚Äî JaCoCo HTML coverage report; deep-link target for the per-test tab
- `testwise.json` ¬∑ `scenarios.json` ¬∑ `flaky.json` ¬∑ `flaky-runs/` ¬∑ `prod-files.txt` ‚Äî sample data
- `tia.db` (gitignored) ‚Äî regenerated by the script
