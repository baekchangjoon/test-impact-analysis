3 · tia impact — diff 기반 테스트 선별

베이스라인 커밋( — tia index로 per-test 커버리지를 DB에 저장한 시점)에 적용하려는 변경(diff)을 비교하고, 그 변경 라인을 커버하는 테스트만 골라낸다.

📌 무엇과 무엇을 비교하나? origin HEAD ↔ local HEAD 같은 고정 비교가 아니다. 베이스라인은 위의 인덱싱된 커밋이고, 비교 대상은 "지금 평가하려는 diff" — 커밋 전 워킹트리, 브랜치 vs 베이스라인, PR diff 등 무엇이든 된다. 이 데모에서는 시나리오마다 대상 파일/라인에 // tia-probe 한 줄을 넣은 합성 diff를 베이스라인과 비교했다(perturb-and-revert — 워킹트리 변경은 즉시 복원). 선별 규칙: diff의 변경 라인 ∩ 각 테스트의 커버 라인 ≠ ∅ → 🎯 정밀 선별.

용어 설명

🎯 정밀 선별
DETERMINISTIC — 변경된 코드 라인이 이 테스트의 커버리지 기록에 정확히 포함되어 있음. 즉 이 변경이 직접 실행하는 테스트.
🛡 보수적 선별
CONSERVATIVE — 변경을 특정 커버리지에 매핑할 수 없어(새 파일·설정 변경 등) 누락을 막기 위해 안전하게 후보로 포함. fallback.
▼ NN% 절감
전체 테스트 중 실행을 건너뛸 수 있는 비율. 높을수록 TIA 이득이 큼.
🛡 safe fallback
보수적 선별로 전체를 포함한 경우 — 절감은 없지만 누락 위험 0.
⚠ 0개 선별
어떤 테스트도 닿지 않는 파일을 변경 → 선별 0개. 커버리지 사각지대(탭 5 참조).
